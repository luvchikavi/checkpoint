2.1 RENEWABLE ENERGY INITIATIVE
--------------------------------------------------------------------------------
Base Scenario (50% renewable adoption):
  Investment:         €12,000,000
  Annual Reduction:   35,000 tons CO₂e
  Payback Period:     6 years
  Cost per ton:       €342/ton

CALCULATION LOGIC:
# Baseline energy consumption from Scope 2
total_energy_consumption = 186,000 tons CO₂e / 0.45 kg CO₂e/kWh
                         = 413,333 MWh/year

# 50% renewable conversion
renewable_portion = 413,333 × 0.50 = 206,667 MWh/year

# Emission reduction
reduction = 206,667 MWh × 0.45 kg CO₂e/kWh × (1 - 0.05)
          = 35,000 tons CO₂e/year
          # (assuming renewables have 5% lifecycle emissions)

# Investment based on PPA/PV installation costs
cost = 206,667 MWh × €60/MWh (levelized cost difference)
     = €12.4M (rounded to €12M)

SLIDER SCALING (Linear interpolation):
if slider_value = 100%:
    reduction = 35,000 × (100/50) = 70,000 tons
    cost = €12M × (100/50) = €24M

REFERENCES:
  - IRENA Renewable Power Generation Costs (2023)
  - EU Green Deal renewable energy targets
  - Typical solar PV LCOE: €40-80/MWh

4. FINANCIAL ANALYSIS - CALCULATION METHODOLOGY
================================================================================

4.1 CARBON PRICING
--------------------------------------------------------------------------------
Default:        €100/ton CO₂e
Range:          €0-500/ton
Reference:      EU ETS spot price 2023-2024 (€80-100 typical range)
Justification:  Internal carbon pricing commonly used by Fortune 500 companies

4.2 NPV CALCULATION
--------------------------------------------------------------------------------
Net Present Value formula:
NPV = -Initial_Investment + Σ(Annual_Savings / (1 + discount_rate)^t)
      for t = 1 to analysis_period

Annual Savings calculation:
Annual_Savings = (CO2_Reduction × Carbon_Price) + Energy_Cost_Savings

Energy Cost Savings (estimate):
Energy_Savings = CO2_Reduction × 0.40 × Electricity_Price / 1000
                # Assumes 40% of CO2 reduction comes from energy efficiency
                # Default electricity price: €120/MWh

DISCOUNT RATE ASSUMPTIONS:
  Default:        5%
  Range:          0-15%
  Justification:  Typical corporate WACC for tech companies: 5-8%

4.3 SIMPLE PAYBACK PERIOD
--------------------------------------------------------------------------------
Simple_Payback = Total_Investment / Annual_Savings

Example:
  Investment:     €12M (renewable energy)
  Annual savings: €3.5M (35,000 tons × €100/ton)
  Payback:        12M / 3.5M = 3.4 years

4.4 IRR (INTERNAL RATE OF RETURN) - APPROXIMATION
--------------------------------------------------------------------------------
Simplified IRR approximation:
IRR ≈ (Annual_Savings / Initial_Investment) × 100

For more accurate IRR, would need to solve:
NPV = 0 = -Investment + Σ(Cash_Flow / (1 + IRR)^t)

5.2 SCOPE CALCULATION LOGIC
--------------------------------------------------------------------------------
def calculate_scope_emissions(historical_data):
    """
    Allocates emissions to GHG Protocol scopes

    Scope 1: Direct emissions from owned/controlled sources
    Scope 2: Indirect emissions from purchased energy
    Scope 3: All other indirect emissions in value chain
    """

    current_month = historical_data.iloc[-1]

    # Scope 1: Minimal for software company
    scope1 = 500  # Fleet + generators + refrigerants

    # Scope 2: All purchased energy
    scope2 = (current_month['Data Centers'] +
              current_month['Cloud Infrastructure'] +
              current_month['Office Buildings'])

    # Scope 3: Commute, travel, supply chain
    scope3 = (current_month['Employee Commute'] +
              current_month['Business Travel'] +
              current_month['Software Development'])

    return scope1, scope2, scope3

5.3 OPERATIONS DECISION TOOL - SLIDER LOGIC
--------------------------------------------------------------------------------
def calculate_initiative_impact(slider_percentage, base_scenario):
    """
    Linear scaling from base scenario

    Parameters:
    - slider_percentage: User input (0-100%)
    - base_scenario: Reference point (e.g., 50% for renewable)

    Returns:
    - Scaled reduction (tons CO₂e)
    - Scaled cost (€)
    """

    scaling_factor = slider_percentage / base_scenario['reference_pct']

    reduction = base_scenario['base_reduction'] × scaling_factor
    cost = base_scenario['base_cost'] × scaling_factor

    # Example: Renewable Energy at 80%
    # scaling_factor = 80 / 50 = 1.6
    # reduction = 35,000 × 1.6 = 56,000 tons
    # cost = €12M × 1.6 = €19.2M

    return reduction, cost

ASSUMPTIONS IN LINEAR SCALING:
  - Economies of scale: Not modeled (real-world shows decreasing marginal costs)
  - Technical feasibility: Assumes 100% adoption is technically possible
  - Temporal factors: Ignores implementation timeline
  - Interactions: Treats initiatives independently (no synergies modeled)

5.4 LCA CALCULATOR - TOTAL IMPACT CALCULATION
--------------------------------------------------------------------------------
def calculate_lca_total(materials, energy, transport):
    """
    Aggregates lifecycle impacts across all stages

    Life Cycle Stages (EN 15804):
    - A1-A3: Product stage (materials + manufacturing)
    - A4: Transport to site
    - B1: Use phase (estimated)
    - C1-C4: End of life (estimated)
    """

    # Sum material impacts
    materials_total = sum(m['quantity'] × m['gwp'] for m in materials)

    # Energy/manufacturing impact
    energy_total = energy['electricity_impact'] + energy['gas_impact'] +
                   energy['diesel_impact'] + energy['heat_impact']

    # Transport impact
    transport_total = transport['inbound'] + transport['outbound']

    # Estimate use phase (simplified: 30% of manufacturing energy)
    use_phase = energy_total × 0.30

    # Estimate end-of-life (simplified: 5% of materials)
    eol = materials_total × 0.05

    # Total product carbon footprint
    total_impact = materials_total + energy_total + transport_total +
                   use_phase + eol

    return total_impact

SIMPLIFICATIONS IN LCA:
  1. Use phase: Crude 30% estimate; should be based on actual product usage
  2. End of life: 5% is generic; real recycling/disposal varies widely
  3. Cutoff rules: Materials <1% not individually tracked
  4. Data quality: Mix of primary (specific) and secondary (database) data
  5. System boundaries: Cradle-to-grave, excludes infrastructure

6.2 LIMITATIONS & CAVEATS
--------------------------------------------------------------------------------

DASHBOARD:
  - Historical data is synthetic (random generation for demo)
  - Baseline emissions are estimates, not audited figures
  - Monthly variations don't reflect actual seasonality

OPERATIONS DECISION TOOL:
  - Linear scaling assumes no diminishing returns
  - Costs are rough estimates, not detailed engineering studies
  - Doesn't account for implementation timelines or phasing
  - Ignores synergies/conflicts between initiatives
  - Payback calculations simplified (no maintenance, degradation, etc.)

LCA CALCULATOR:
  - Database values are global averages, not product-specific
  - Use phase and EOL are crude estimates
  - Doesn't include capital equipment, infrastructure
  - No uncertainty propagation through calculations

FINANCIAL ANALYSIS:
  - NPV uses constant cash flows (no inflation, degradation)
  - IRR is approximated, not solved precisely
  - Doesn't include financing costs, tax implications
  - Sensitivity only tests 2 variables (carbon price, discount rate)
